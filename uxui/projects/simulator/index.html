The Wealth Disparity Project: Exploring Economic Inequality

Project Overview

View Live Project: The Wealth Disparity Project

The wealth disparity project aims to correct the misconception that poverty results from poor decision-making or laziness. The project seeks to educate the general public about the worsening homelessness problem in the US by exploring the impacts of wealth disparity.

Goals and Objectives

  • Raise Awareness: Illustrate how inequality affects everyday life.
  • Challenge Assumptions: Dispel stereotypes around poverty.
  • Interactive Experience: Let users simulate economic futures through an interactive web experience.

My Role

I led the design and development of this interactive project. My responsibilities included:

  • Front-end & back-end development, UX/UI and visual design.
  • Research on wealth disparity.
  • Prototyping each of the economic scenario.
  • Designing user interfaces that engage users and promote awareness.
  • Conducting user testing and refining design based on feedback.

Tools and Technologies Used

  • Visual Design: Figma, Photoshop, Illustrator, Adobe Color
  • Development: Visual Studio Code, D3.js, Back4app, noUiSlider
  • Additional Technologies: Hand sketches, Canva

User Research and Findings

Summary of User Research

Research included interviews, surveys, and studying existing literature on economic inequality. Key insights revealed that many people are unaware of the extent and impact of wealth disparity, with common misconceptions attributing poverty to laziness or poor decision-making. Interactive and visual content proved more effective in conveying complex economic concepts.

Usability Testing

Testing Methodology

In-person moderated testing with 2 volunteer.

Tasks and Scenarios

  • Task 1: Navigate to the Simulator Page - Users were asked to navigate from the intro page to the interactive simulator page.
  • Task 2: Interact with the Simulator - Users were instructed to interact with four sliders (Tax Rate, Minimum Wage, Social Welfare, and Market Forces) and observe changes in the image section.
  • Task 3: Leave a Comment - Users were asked to leave a comment in the comment section of the Simulator Page.

Data Collection

Metrics measured included task completion rates, time on task, and user satisfaction ratings. Observation techniques involved note-taking by the moderator.

Test Facilitation

Participants were briefed on the project and instructed to complete tasks while verbalizing their thoughts and actions. The moderator guided participants through tasks, answered questions, and took notes on observations and feedback.

Analysis and Reporting

Qualitative analysis was performed on observations and feedback, resulting in a summary report highlighting key findings, user comments, and recommendations.

Recommendations and Action Items

  • Improve image transitions in the simulator for a smoother experience.
  • Clarify the purpose of the comment section or move it to a separate page.
  • Enhance font and layout of the intro page for better accessibility and readability.
  • Add a character or guide to make the design more relatable.
  • Shorten the intro page content for conciseness.
  • Enhance visibility of the image description on the simulator page.
  • Add a "back" button to allow users to return to the intro page from the simulator page.

Design Decisions (Based on Feedback)

  • Converted the comment section into a pop-up.
  • Streamlined intro content with background animation.
  • Repositioned image description above sliders.
  • Added a “Back” button to the simulator.
  • Created a collapsible “Data and Facts” sidebar.

Challenges and Solutions

  • Challenge: Visual Complexity
    • Solution: Built a flexible visual system to handle 16+ economic scenarios
  • Challenge: Implementing different libraries for interactive elements.
    • Solution: Combined noUiSlider with D3.js and Back4App
  • Challenge: Data Overload.
    • Solution: Used layout clarity and visuals to break up heavy content

Outcomes and Impact

The project was showcased at a design event where it received strong feedback from professionals and peers for its clarity and educational value. Users appreciated the ability to visualize policy impacts in a digestible way.
