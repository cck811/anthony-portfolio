12 Points Shooter – Interactive JavaScript Game

Project Demo: Link

Game Overview

12 Point Shooter is a fast-paced 2-player web game designed for browsers using HTML, CSS, and JavaScript. Players take turns selecting cards to accumulate points, while dodging hidden penalties. Strategic risk-taking is introduced through a “Test My Luck” button, allowing players to pick twice per turn, but with the risk of losing it all. The game combines user-friendly interaction design, animation, and audio to create an engaging digital experience.

Game Rules

In 12 Point Shooter, players take turns picking cards to reach 12 points first. Most cards add points, but some subtract or reset your score. You can test your luck once per turn to pick two cards instead of one. After each turn, play switches automatically. The first to 12 wins.

Design Process

Before development began, I mapped out the game logic with a flowchart to visualize the gameplay sequence and player decisions. I also created a detailed UI layout comp to guide interface placement and ensure responsiveness across screen sizes. The chosen color palette was selected for contrast, readability, and visual harmony to match the name of the game.

Development & Features

The game logic is built using HTML, CSS, and JavaScript with a clean modular structure. Two players take turns picking from six shuffled cards, each with different effects. Some cards reward points, while others punish by subtracting or resetting the score. A “Test My Luck” button added a risk factor to the game.

Alongside the core mechanics, I implemented:

  • Card Shuffling: Randomly reassigns values after each turn to prevent predictability.
  • Sound Effects: Custom gunshot sound on every card pick, and a celebratory jingle when the game is won.
  • Click Visual Feedback: Clicking anywhere on the screen triggers a gunshot sound and shows a temporary bullet-hole effect at the clicked location for immersive feedback.
  • Winning Screen: A full-screen overlay with custom messaging appears when a player reaches 12 points.
  • Cowboy Character: A stylized cartoon cowboy illustration was added in the final version to add visual charm and personality to the interface.

User Feedback

The initial prototype focused on testing the core game mechanics: six randomized buttons, turn-based scorekeeping for two players, and a basic UI. This version was intentionally minimal to validate the game logic. I shared the prototype with a small group of users and gathered helpful feedback that informed the final iteration.

These insights led me to add a ‘Test My Luck’ button for risk-based decisions, enhance the UI with a Western-themed layout and cartoon character, implement custom gunshot and win sounds, and create a click animation effect resembling a bullet hole to match the game’s style and improve engagement.

Challenges & Learnings

One of the biggest challenges was creating a balanced game mechanic that felt both fair and engaging. Translating game logic into functional JavaScript—especially managing turn logic, score tracking, and risk dynamics—required precise event handling and careful debugging.

I also learned how important sound and visual feedback are to the game experience. Implementing dynamic UI elements like modals, animations, and clickable effects pushed my understanding of how front-end interactivity can heighten user engagement.

Reflections

This project deepened my appreciation for the synergy between design and development. What began as a simple logic test evolved into a fully themed, interactive web game that leverages both visual storytelling and UX principles.

I realized that even small enhancements—like the sound effects or “Test My Luck” mechanic—can significantly improve playability and immersion. More importantly, I saw how early prototyping and user feedback are crucial to refining a concept into a delightful experience.
